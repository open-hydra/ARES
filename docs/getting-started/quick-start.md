# Quick Start: Turbulent Flat Plate

This guide walks you through running the **turbulent flat plate** test case with the Menter SST model — a canonical RANS verification whose skin friction is compared against the **NASA Turbulence Modeling Resource (TMR)**. It exercises the full ARES pipeline: a real-fluid table, a turbulent Navier–Stokes run, and a wall boundary condition.

!!! note "Prerequisites"
    - ARES built and available at `bin/ARES` (see [Installation](installation.md))
    - Python 3 with NumPy and Matplotlib (for post-run verification)

---

## Physics Overview

A uniform stream flows over a flat plate aligned with the flow, developing a zero-pressure-gradient **turbulent** boundary layer. The standard validation quantity is the skin-friction coefficient $c_f(x)$, which for an equilibrium turbulent boundary layer follows a well-established law that the NASA TMR tabulates for each turbulence model.

**Parameters** (as configured in the bundled case):

| Parameter | Value | Notes |
|-----------|-------|-------|
| Freestream velocity | $u_\infty = 100$ m/s | |
| Static pressure | $p = 1.0\times10^{5}$ Pa | |
| Static temperature | $T = 555$ K | |
| Working fluid | air (real-fluid table) | $(p,h)$ table generated by FLINT |
| Turbulence model | Menter SST | `turbulence-model = SST` |
| Regime | turbulent Navier–Stokes | `simulation-type = turbulent` |

**Expected result:** the computed $c_f(x)$ collapses onto the NASA TMR reference in $\mathrm{Re}_x$ space.

## Case Directory Structure

The case lives at `test/Flat_Plate_SST/`:

```
Flat_Plate_SST/
├── input.ini                 # Solver + pre-processing configuration
├── ARES.sh                   # Run script (compile / solve / kill)
├── MESH/mesh.tec             # Case mesh (bundled; generated by ATLAS [GRIB])
├── INPUT/                    # Case data produced by ATLAS (not versioned)
│   ├── phase.txt             #   real-fluid table descriptor
│   ├── thermo.dat            #   tabulated thermodynamic data
│   ├── ic.tec                #   initial condition (mesh + field)
│   └── bc.txt                #   boundary-condition table
├── reference/                # NASA TMR reference data (cf.dat, kw.dat, mut.dat)
├── set_constant_cp.py        # Flatten the table to constant cp (for the analytic comparison)
├── set_constant_transport.py
├── validate_sst.py           # Verification script
└── OUTPUT/                   # (created at runtime)
```

!!! info "Where do the INPUT files come from?"
    `phase.txt`, the tables, `ic.tec`, and `bc.txt` are produced by **ATLAS** from the `[GPB-*]`, `[ICB-*]`, and `[BCB-*]` sections of `input.ini` (the bundled `MESH/mesh.tec` comes from its `[GRIB-*]` blocks). ATLAS is distributed separately and must be run once before the first solve — only the mesh and the reference data are versioned with the case. See the [User Guide](../user/using.md) for the workflow and file formats.

## Solver Configuration

The `input.ini` mixes the ATLAS pre-processing blocks with the ARES solver blocks. The solver-relevant part is:

```ini
[ARES-Parameters]
simulation-type = turbulent     # Turbulent Navier–Stokes
iter-threshold  = 700000        # Maximum iterations
newrun          = .true.        # Start from the initial condition

[ARES-Numerics]
cfl                   = 0.6
vnn                   = 0.6      # Viscous (von Neumann) stability number
time-scheme           = RK3      # 3-stage SSP Runge–Kutta
time-accurate         = false    # March to steady state
integration-variables = prim     # Primitive-variable update
space-reconstruction  = MUSCL    # 2nd-order reconstruction
flux-limiter          = vanleer  # van Leer slope limiter
riemann-solver        = HLLC     # HLLC numerical flux

[ARES-IO]
sol-diter     = 50000
shell-diter   = 1000             # Print residual every 1000 iters
sol-variables = transport        # Also write μ, k, μt

[ARES-RANS]
turbulence-model = SST           # Menter SST k–ω
Prt = 0.85
```

The real-fluid table is requested in the ATLAS block:

```ini
[GPB-Phase1]
type  = real-fluid
name  = air
fluid = air
pmin  = 0.50e5    ; minimum pressure [Pa]
pmax  = 2.0e5     ; maximum pressure [Pa]
Tmin  = 500.0     ; minimum temperature [K]
Tmax  = 600.0     ; maximum temperature [K]
NP    = 200       ; pressure grid points
NH    = 200       ; enthalpy grid points
```

The inflow carries freestream turbulence values (`kappa` = $k$, `omega` = $\omega$) in the `[ICB-Block1]` block, as required by the two-equation model.

## Running the Simulation

Navigate to the case directory and run:

```bash
cd test/Flat_Plate_SST
./ARES.sh solve
```

The `ARES.sh solve` task:

1. Creates `OUTPUT/` and `bin/` if needed.
2. Sets `ulimit -s unlimited` and `KMP_STACKSIZE` (ARES uses a large stack).
3. Launches the solver binary.

**Parallel execution** — run with N OpenMP threads:

```bash
./ARES.sh solve -p 4        # 4 OpenMP threads
./ARES.sh solve -m 2 -p 4   # 2 MPI processes × 4 OpenMP threads (hybrid)
./ARES.sh solve -b -p 4     # background; output redirected to ./logfile
```

!!! tip "Loading checks"
    If any input is missing or malformed, ARES prints an error and exits, so a clean `Loading … OK` block is a good sign that the case is set up correctly.

During execution, ARES prints the iteration count and the **pressure residual** (the residual monitored for real-fluid runs):

```
ARES | Iter =     1000 | Global iter =     1000 | Pressure residual = 0.264793E-04
ARES | Iter =     2000 | Global iter =     2000 | Pressure residual = 0.262362E-04

```

### Output files

| File | Content |
|------|---------|
| `OUTPUT/field.tec` | Full 2-D field ($p$, $u$, $v$, $h$, $T$, $k$, $\omega$, $\mu_t$, …) at convergence |
| `OUTPUT/wall.tec` | Wall quantities (skin friction, $y^+$) |
| `OUTPUT/residual-history.dat` | Convergence history |

## Post-Run Verification

After the run, compare the computed skin friction to the NASA TMR reference:

```bash
python3 validate_sst.py
```

The script reads `OUTPUT/field.tec` and `OUTPUT/wall.tec`, computes $c_f(x)$ and the wall-scaled $k^+,\omega^+$ profiles, overlays them on the NASA TMR data in `reference/`, and prints the RMS / max relative $c_f$ error with a **PASS/FAIL** verdict (PASS for RMS < 5 %).

!!! tip "Constant-property comparison"
    The NASA reference assumes a calorically simple gas. Run `set_constant_cp.py` and `set_constant_transport.py` first to flatten the real-fluid table to constant $c_p$ / transport, so the comparison is exact rather than approximate.
