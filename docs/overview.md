# Overview

ARES (**A** **R**eal-fluid **E**quations **S**olver) is an open-source compressible-flow solver for the Euler and Navier–Stokes equations on multi-block structured grids, written in modern Fortran. It targets compressible flows in which the working fluid departs from the ideal-gas model — its thermodynamic and transport properties are evaluated from tabulated **real-fluid** data — and it includes low-Mach preconditioning so that the same density-based scheme remains accurate from nearly incompressible to hypersonic regimes.

---

## Hydra CFD Suite

ARES is the **real-fluid compressible-flow solver** of the **Hydra** CFD ecosystem — an integrated suite of tools for multi-physics simulation of complex systems.

| Component | Role | Status |
|-----------|------|--------|
| [**ATLAS**](https://github.com/open-hydra/ATLAS) | Pre-processor: mesh generation, initial & boundary conditions, real-fluid thermodynamic tables | Separate package |
| **ARES** | Solver: compressible Euler / Navier–Stokes for real fluids | This package |
| [**FUSS**](https://github.com/open-hydra/FUSS) | Solid heat-conduction solver | Separate package |
| [**GROOT**](https://github.com/open-hydra/GROOT) | Radiative-transfer solver | Separate package |

!!! info "ARES and ATLAS"
    The data ARES consumes (mesh, initial conditions, boundary conditions, real-fluid property tables) is **produced by ATLAS**. In the Hydra workflow, a single `input.ini` drives both: ATLAS reads its own `[GRIB-*]`, `[GPB-*]`, `[ICB-*]` and `[BCB-*]` sections to generate the case data in `INPUT/`, and ARES reads its `[ARES-*]` sections to solve. See the [User Guide](user/using.md) for the file formats if you prepare the input manually.

## ARES Capabilities

ARES solves the compressible Navier–Stokes equations in conservative form:

$$
\frac{\partial \mathbf{U}}{\partial t}
+ \nabla\!\cdot\!\mathbf{F}^{c}(\mathbf{U})
= \nabla\!\cdot\!\mathbf{F}^{v}(\mathbf{U})
+ \mathbf{S}
$$

where $\mathbf{U}$ is the conservative-variable vector, $\mathbf{F}^c$ the convective fluxes, $\mathbf{F}^v$ the viscous fluxes, and $\mathbf{S}$ the source terms (turbulence, rotating frame). Both the inviscid limit (Euler equations) and the full viscous form (Navier–Stokes) are supported. The simulation regime is selected with a single switch, `simulation-type` = `euler`, `laminar`, or `turbulent`.

## Physical Models

### Real-fluid thermodynamics

ARES evaluates thermodynamic and transport properties from **tabulated real-fluid data** generated by the [FLINT](https://github.com/MarcoGrossi92/FLINT) library. Properties are stored on a structured **pressure–enthalpy $(p,h)$ grid** and retrieved by bilinear interpolation, so the equation of state is *not* restricted to the ideal-gas law $p=\rho R T$: it reproduces the real-fluid behaviour of the chosen fluid over the tabulated range.

| Property | Model |
|----------|-------|
| Equation of state | Tabulated real-fluid $(p,h)$ tables (FLINT) |
| Density, temperature, speed of sound | Interpolated from the $(p,h)$ table |
| Specific heat, enthalpy | Interpolated from the $(p,h)$ table |
| Dynamic viscosity $\mu(p,h)$ | Interpolated from the transport table |
| Thermal conductivity $\lambda(p,h)$ | Interpolated from the transport table |
| Back-end | Reference data from CoolProp / NASA correlations (via ATLAS/FLINT) |

See [Real-Fluid Thermodynamics](theory/thermo.md) for details.

### Turbulence

All turbulence models are RANS eddy-viscosity (or Reynolds-stress) closures selected at run time via the `[ARES-RANS]` section. The Boussinesq hypothesis links the Reynolds-stress tensor to the mean strain rate via the turbulent viscosity $\mu_t$; the SSG-LRR model instead transports the Reynolds stresses directly.

| Model | Type | Key feature |
|-------|------|-------------|
| Spalart–Allmaras (SA) | 1-equation | Baseline; R / RC rotation–curvature corrections |
| SA-QCR2000 | 1-eq + algebraic | Quadratic Constitutive Relation for stress anisotropy |
| SA-rough | 1-equation | Sand-grain wall-roughness extension |
| SAcomp | 1-equation | Compressibility correction |
| Menter SST | 2-equation | $k$–$\omega$ / $k$–$\varepsilon$ blending; robust in adverse pressure gradients |
| Wilcox $k$–$\omega$ 2006 | 2-equation | Updated cross-diffusion; improved freestream behaviour |
| SSG-LRR | Reynolds-stress (7-eq) | Speziale–Sarkar–Gatski / Launder–Reece–Rodi |

### Low-Mach preconditioning

Density-based schemes lose accuracy and stall in convergence as $M \to 0$, because the disparity between the acoustic and convective wave speeds makes the system stiff. ARES applies a **Weiss–Smith-type preconditioner** to the time derivative, rescaling the acoustic eigenvalues toward the convective speed. It is activated by setting `integration-variables = prec` in `[ARES-Numerics]`, which also enables the preconditioned Riemann solvers (`PLLF`, `HLLC Prec`).

See [Low-Mach Preconditioning](theory/preconditioning.md).

## Numerical Methods

### Spatial discretisation

| | Details |
|-|---------|
| Framework | Cell-centred finite volume on structured multi-block hexahedral grids |
| Reconstruction | MUSCL / MUSCL-SD — first- and second-order accurate |
| Flux limiters | van Leer · van Albada · minmod · MC · superbee · LIMO3 |
| Riemann solvers | Rusanov · PLLF · HLLE · HLLC · HLLC Prec · HLLC Rotated |
| Diffusive fluxes | Face-metric tensor mapping with a wide gradient stencil |

### Time integration

| | Details |
|-|---------|
| Explicit | Multi-stage SSP Runge–Kutta (Euler / RK2 / RK3) |
| Convergence acceleration | Implicit Residual Smoothing (IRS), geometric multigrid |
| Low-Mach | Weiss–Smith preconditioning of the pseudo-time derivative |

### Parallel computing

| Mode | Details |
|------|---------|
| Shared memory | OpenMP thread-level parallelism within a block |
| Distributed memory | MPI domain decomposition across blocks |
| Hybrid | OpenMP + MPI combined runs on HPC clusters |
