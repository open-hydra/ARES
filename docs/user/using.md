# Running ARES

This page describes the complete workflow for running an ARES simulation: preparing the case directory, defining the input, launching the solver, and inspecting the output.

A case is driven by:

- `input.ini` — the main INI file. It contains both the **ARES solver** sections (`[ARES-*]`) and the **ATLAS pre-processing** sections (`[GRIB-*]`, `[GPB-*]`, `[ICB-*]`, `[BCB-*]`).
- `INPUT/` — the supporting data files generated by ATLAS from those pre-processing sections: the real-fluid table (`phase.txt`, `thermo.dat`), the initial condition (`ic.*`), and the boundary conditions (`bc.txt`).

!!! tip
    The files in `INPUT/` are normally built by **ATLAS**. ATLAS is powerful but not strictly required: the files can be created manually or by custom scripts as long as they follow the expected format (see [Initial Conditions](initial-conditions.md) and [Boundary Conditions](boundary-conditions.md)).

---

## Simulation Workflow

A typical ARES run follows four phases:

1. **Setup** — define the mesh, real-fluid table, initial and boundary conditions (the `[GRIB/GPB/ICB/BCB]` blocks of `input.ini`).
2. **Preprocess** — ATLAS generates the `INPUT/` data from those blocks.
3. **Solve** — ARES advances the solution in time (explicit integration).
4. **Postprocess** — solution fields, wall data, probes, and the residual history are written to `OUTPUT/`.

---

## Case Directory Structure

Every ARES case follows a standard layout:

```
my_case/
├── input.ini                 ← case configuration (ARES + ATLAS blocks)
├── ARES.sh                   ← convenience run script
├── INPUT/
│   ├── phase.txt             ← real-fluid table descriptor
│   ├── thermo.dat            ← tabulated thermodynamic / transport data
│   ├── ic.*                  ← initial condition (mesh + flow field)
│   └── bc.txt                ← face-by-face boundary-condition table
└── OUTPUT/                   ← created at runtime
    ├── field.*               ← solution fields (when sol-overwrite = true)
    ├── field1.*, field2.*    ← solution snapshots (when sol-overwrite = false)
    ├── wall.*                ← wall quantities (viscous cases only)
    ├── residual-history.dat  ← residual norm vs. iteration
    └── <probe>.txt           ← time-history at probe locations
```

!!! warning
    ARES reads the grid geometry directly from the initial-condition file (the `x`, `y`, `z` node coordinates). There is no separate mesh file consumed by the solver.

## Running the Solver

The bundled `ARES.sh` script wraps the common tasks:

```bash
./ARES.sh compile          # build a local bin/ARES from the master build
./ARES.sh solve            # run in the foreground
./ARES.sh solve -p 4       # run with 4 OpenMP threads
./ARES.sh solve -m 2 -p 4  # run with 2 MPI processes × 4 OpenMP threads (hybrid)
./ARES.sh solve -b -p 4    # run in background (stdout → logfile, PID → .ID)
./ARES.sh kill             # kill a background run via the stored PID
```

`ARES.sh solve` sets the environment ARES needs and launches the binary:

```bash
ulimit -s unlimited        # ARES uses a large stack
export KMP_STACKSIZE=100M  # Intel/OpenMP stack per thread
export OMP_NUM_THREADS=4
./bin/ARES
```

!!! warning "Stack size"
    ARES allocates sizeable automatic arrays. Always run `ulimit -s unlimited` (and set `KMP_STACKSIZE` for Intel/OpenMP) before launching, or the solver may crash with a segmentation fault on the first flux evaluation. `ARES.sh` does this for you.

## Logging and Diagnostics

ARES prints diagnostics to the console (or to `logfile` when run with `-b`). The output begins with a banner, then a **Loading** block, a **Set-up** summary, the **Physical model** and **Numerical scheme** descriptions, and finally the iteration log.

```

 =========================================================================================
 Loading
 =========================================================================================
   Thermodynamics                 OK
   Transport                      OK
   Thermo inversion               OK
   Input file                     OK
   Initial conditions             OK
   Boundary conditions            OK
 =========================================================================================

 =========================================================================================

 =========================================================================================

   Blocks                         1
   Cells                          13056
   Boundary faces                 26576

 Boundary conditions
   Viscous wall                   112
   Inflow/outflow                 192
   Symmetry                       160
 =========================================================================================

 =========================================================================================

 =========================================================================================
 Real Fluid model
   Equations                      Navier-Stokes
   Viscous model                  Turbulent, SST
   Equation of state              Real
   Thermodynamics                 Real fluid
 =========================================================================================

ARES | Iter =     1000 | Global iter =   501000 | Pressure residual = 0.264793E-04
ARES | Iter =     2000 | Global iter =   502000 | Pressure residual = 0.262362E-04
ARES | Iter =     3000 | Global iter =   503000 | Pressure residual = 0.259158E-04
```

!!! tip "Loading checks"
    A clean `Loading … OK` block means the input files were read successfully. If something is missing or malformed, ARES prints an error and exits.

!!! note "Which residual?"
    For real-fluid / low-Mach runs ARES monitors the **pressure** residual (the natural convergence indicator when the primitive set is $(p, \mathbf{u}, h)$ and the pseudo-time derivative is preconditioned). `Global iter` is the cumulative iteration count across restarts; `Iter` is the count within the current run.

## Output

After a run, the `OUTPUT/` directory contains:

| File | Description |
|------|-------------|
| `field.*` | Solution fields ($p$, $u$, $v$, $w$, $T$, $\rho$, $h$, …) |
| `residual-history.dat` | Iteration-by-iteration residual norms |
| `wall.*` | Wall quantities (skin friction, heat flux) — viscous cases only |
| `<probe>.txt` | Time-history at the probe named `<probe>` |

### I/O formats

Controlled by `sol-format` in `[ARES-IO]`:

| Value | Format |
|-------|--------|
| `tecplot ascii` | Tecplot ASCII (`.tec`) — default |
| `tecplot binary` | Tecplot binary (`.szplt`) — requires TecIO |
| `vtk ascii` | VTK ASCII (`.vts`) |
| `vtk raw` | VTK binary (`.vts`) |

### Output variables

By default ARES writes primitive variables ($p$, $u$, $v$, $w$, $T$). Extra groups are enabled via `sol-variables` in `[ARES-IO]`:

- `thermo` — speed of sound, $\gamma$, gas constant, density
- `transport` — laminar viscosity $\mu_\ell$, conductivity $k_\ell$, turbulent viscosity $\mu_t$

Wall output is controlled by `wall-variables`: $y^+$ is always written first; `mech` adds the wall shear components and pressure (`tauX tauY tauZ pw`), `thermal` adds wall temperature and heat flux (`Tw qw`). The sand-grain roughness of the wall cell (`hs`) is always written as the last wall variable (zero on smooth walls).

### Probes

Point-measurement time histories are configured with probe sections:

```ini
[ARES-Probes]
probe1 = exit

[exit]
variables = p T u
index-position = 50 33 1 1   ; i j k block
diter = 100
```

Each probe writes `OUTPUT/exit.txt` with columns for time/iteration and the requested variables.

---

## Restart

Set `newrun = .false.` in `[ARES-Parameters]` to resume. ARES reads the initial state from the latest solution file in `OUTPUT/` instead of `INPUT/`, restoring the flow field, physical time, and the global iteration counter.
