ARES is an open-source compressible-flow solver for the Euler and Navier–Stokes equations on multi-block structured grids, written in modern Fortran. It is built specifically for **real-fluid** flows: thermodynamic and transport properties are evaluated from tabulated real-fluid data, and low-Mach preconditioning keeps the same density-based scheme accurate from nearly incompressible to hypersonic regimes.

ARES is the real-fluid solver of the **[Hydra](https://github.com/open-hydra)** CFD suite. The mesh, initial/boundary conditions, and real-fluid property tables it consumes are produced by the companion pre-processor **[ATLAS](https://github.com/open-hydra/ATLAS)**.

## Features

- **Compressible Euler & Navier–Stokes** — steady-state and time-accurate computations on multi-block structured grids. Regime selected by a single switch: `euler`, `laminar`, or `turbulent`.
- **Real-fluid thermodynamics** — thermodynamic and transport properties interpolated from tabulated pressure–enthalpy $(p,h)$ data (via FLINT); the equation of state is the tabulated real-fluid data itself, not an analytic relation.
- **Low-Mach preconditioning** — Weiss–Smith-type preconditioner with dedicated preconditioned Riemann solvers for accurate, well-converged low-Mach computations.
- **High-order numerics** — second-order MUSCL reconstruction with multiple flux limiters (van Leer, van Albada, minmod, MC, superbee, LIMO3) and Riemann solvers (Rusanov, PLLF, HLLE, HLLC, preconditioned and rotated HLLC).
- **Turbulence modeling** — RANS closures: Spalart–Allmaras (with rotation/curvature, QCR2000, roughness and compressibility variants), Menter SST, Wilcox k-ω 2006, and the SSG-LRR Reynolds-stress model.
- **Convergence acceleration** — Implicit Residual Smoothing (IRS) and geometric multigrid.
- **Parallel execution** — shared-memory parallelism via OpenMP, distributed-memory via MPI, and hybrid OpenMP+MPI runs.
- **Flexible I/O** — solution output in Tecplot (ASCII and binary via TecIO) and VTK formats, point probes for time-history recording, and restart capability.

## Quick Start

### Prerequisites

| Requirement | Details |
|---|---|
| **CMake** | ≥ 3.23 |
| **Fortran compiler** | GNU (`gfortran` 11+) or Intel/oneAPI (`ifort` / `ifx`) |
| **C/C++ compiler** | Required for the GCC LTO-aware archiver and optional components (TecIO) |

### Build

```bash
git clone https://github.com/open-hydra/ARES.git
cd ARES

# initialise the submodule dependencies
./install.sh update

# Build with GNU compilers and OpenMP
./install.sh build --compilers=gnu --use-openmp

# — or with Intel compilers and a fuller feature set —
./install.sh build --compilers=intel --use-openmp --use-mpi --use-tecio
```

The executable is placed in `bin/ARES`.

See the [Installation Guide](https://open-hydra.github.io/ARES/getting-started/installation/) for all build options, CMake presets, and troubleshooting.

### Run the Turbulent Flat Plate

```bash
cd test/Flat_Plate_SST
./ARES.sh solve
```

> **Note** — ARES consumes case data (real-fluid tables, initial and boundary conditions) generated by **[ATLAS](https://github.com/open-hydra/ATLAS)**. Only the mesh and reference data are versioned with each case; ATLAS must be run once to produce the `INPUT/` files before the first solve.

See the [Quick Start](https://open-hydra.github.io/ARES/getting-started/quick-start/) for a full walkthrough.

## Dependencies

ARES is built on top of several companion libraries, included as Git submodules:

| Library | Role |
|---|---|
| [FLINT](https://github.com/MarcoGrossi92/FLINT) | Real-fluid thermodynamic & transport tables |
| [ORION](https://github.com/MarcoGrossi92/ORION) | Multi-format I/O (Tecplot, VTK, Plot3D) |
| [FiNeR](https://github.com/szaghi/FiNeR) | INI configuration file parser |

Optional external libraries: **OpenMP**, **MPI**, **TecIO**.

Case data (mesh, initial/boundary conditions, property tables) is produced by the companion pre-processor **[ATLAS](https://github.com/open-hydra/ATLAS)**, distributed separately.

## Project Structure

```
ARES/
├── src/
│   ├── app/           # Main application (solver entry point + doc generator)
│   ├── lib/           # Solver library (base, config, numerics, physics, io, parallel, driver, diagnostic)
│   └── test/          # Unit-test scaffold
├── lib/               # Git submodule dependencies (FLINT, ORION, FiNeR)
├── test/              # Verification & validation cases
│   ├── Flat_Plate_SA/        #   turbulent flat plate — Spalart–Allmaras
│   ├── Flat_Plate_SST/       #   turbulent flat plate — Menter SST
│   ├── Flat_Plate_Wilcox2006/#   turbulent flat plate — Wilcox k-ω 2006
│   ├── Flat_Plate_SGGLRR/    #   turbulent flat plate — SSG-LRR
│   ├── HTD/                  #   supercritical heat transfer (real-fluid)
│   └── Prt-correction/       #   turbulent Prandtl correction
├── docs/              # MkDocs documentation source
├── cmake/             # CMake modules
├── install.sh         # Build helper script
├── CMakePresets.json
└── CMakeLists.txt
```
